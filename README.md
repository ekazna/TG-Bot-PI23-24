# Telegram-бот для малого бизнеса

Учебный командный проект по дисциплине **«Программная инженерия»**.

Проект представляет собой прототип Telegram-бота для малого бизнеса — салона маникюра или частного мастера. Бот помогает автоматизировать запись клиентов: клиент может выбрать услугу, мастера, дату и время, а мастер или администратор — управлять расписанием, услугами и записями.

Основная ценность проекта — в прохождении полного цикла разработки программного продукта: от исследования предметной области и сбора требований до проектирования, реализации прототипа, тестирования и подготовки документации.

## Пользовательские роли

### Клиент

Клиент может:

* просматривать услуги и цены;
* выбирать мастера;
* смотреть информацию о мастере;
* выбирать дату через календарь;
* выбирать свободный временной слот;
* подтверждать запись;
* просматривать свои активные записи;
* отменять запись;
* получать информацию о салоне.

### Мастер / администратор

Мастер или администратор может:

* просматривать расписание;
* просматривать записи на выбранную дату;
* управлять временными слотами;
* отменять записи;
* редактировать информацию о себе;
* просматривать список услуг;
* добавлять и удалять услуги.

## Реализованный функционал

В прототипе были реализованы основные сценарии для клиента и мастера:

* просмотр услуг и цен;
* выбор мастера;
* просмотр информации о мастере;
* выбор даты через календарь;
* выбор временного слота;
* создание и подтверждение записи;
* просмотр записей;
* отмена записи;
* управление расписанием;
* добавление и удаление услуг.

Отдельно была адаптирована библиотека для работы с датами, чтобы календарь корректно использовался внутри сценария записи.

## Тестирование

Для проекта были подготовлены позитивные и негативные тестовые сценарии.

Пример основного пользовательского сценария:

1. Клиент открывает меню бота.
2. Выбирает раздел с мастерами.
3. Просматривает информацию о мастере.
4. Выбирает услугу.
5. Выбирает дату через календарь.
6. Выбирает доступный временной слот.
7. Подтверждает запись.
8. Получает сообщение об успешной записи.

Также были описаны возможные ошибки: отсутствие свободных дат, отсутствие доступных временных интервалов, ошибка подтверждения записи и некорректная работа кнопок.
